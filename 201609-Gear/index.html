コロンブス日和 - Gear

大規模データの検索と閲覧

前回は大規模な階層情報をズーミング検索する DragZoomというシステムを紹介しました。 DragZoomは パソコンやタブレット上の マウスや指の操作で ズーミングを行なうことによって大規模データの検索を行なうものですが、 いつでもどこでも計算機を使うユビキタス環境のどこでも ポインティングデバイスが使えるとは限りません。 ポインティングデバイスが使えない環境でも 大きな階層情報をうまく検索できると便利です。 今回は、ボタンを二つ使うだけで大規模な階層データを検索できる 「Gear」というシステムを紹介します。

段階的な階層データのナビゲーション

DragZoomのように 階層データをなめらかにズーミングして検索する方法はこれまで 沢山研究されてきているのですが、 デスクトップやブラウザ上で広く利用されてる例はほとんど無いようです。 例えばMacのデスクトップ画面でファイルを操作するプログラム(ファインダ)では、 ファイルの階層構造を視覚化/ナビゲーションする方法がいくつか用意されていますが、 マウスやキーを操作して階層構造の親子リンクを段階的にたどっていくことにより 階層型ファイルシステムのナビゲーションを行なうようになっています。

たとえばひとつ下の階層のフォルダの内容を見たい場合、 そのフォルダをマウスでダブルクリックして開いてみたり、 キーボードを使って選択行を移動してから選択するという方法がよく使われています。

パソコンのハードディスクの中には意外と沢山のファイルが含まれています。 自分のMacBook Proを調べてみると 300万個のファイルが含まれていました。 私はファイルを沢山持っている方かもしれませんが、 普通のユーザでも100万個ぐらいのファイルが含まれているようです。 現在のパソコンのファイルはすべて階層的に管理されており、 ほぼ全てにファインダなどからアクセスできるようになっています。 100万個のファイルの中からひとつを選ぶのは 相当大変だろうと思われますが、 実際のパソコン操作においてそれほど苦労している人は多くないでしょう。 大量の情報から検索を行なうときは 沢山の操作をしなければならないと考えがちですが、 きちんと階層的に管理されている場合、 それほど沢山の操作をしなくても必要な情報をみつけることができるのが普通です。 実際、パソコンのデスクトップ画面でマウスを使って 階層のトップから階層をたどって特定のファイルをみつけることは それほど大変作業ではありませんが、 その時の操作はせいぜい10回以下です。

Akinator というサービスを使うと、 2択の質問に10回程度答えるだけで 自分が頭に思っている人物を特定されてしまいます。

こういう事情なので、 キーボードを使って段階的に階層を絞り込みつつ 検索を行なう方法が実用的に広く使われています。

ふたつのキーだけで階層構造をナビゲーション

このように4個のキーを使って階層データのナビゲーションを行なうことは よくありますが、実は キーをふたつしか使わなくても同様のナビゲーションを行なうことが可能です。 そのような「Gear」というシステムを紹介します。

Gear

Gearでは▲と▼というふたつのキーだけを 利用してナビゲーションを行ないます。 Gearで「店リスト」を表示すると， ファインダの場合と同じようなリストが表示されます。 ▼を3回押すと 図\ref{screenshot42}のように「食料品店」が選択されるが， そこで操作を中断して一定時間待つと「食料品店」の下位層が自動的に展開されて， 図\ref{screenshot7}のようにその最初の要素が選択される．
\caption{「食料品店」の下位階層を自動展開} \label{screenshot7} \end{figure}

ここで▼を2回押して「生鮮食料品店」を選択したまま一定時間待つと， 図\ref{screenshot8}のように 下位層が自動的に展開され，最初の要素である「魚屋」が選択される．

つまり，▶のようなキーを押さなくても， 一定時間待つことによって同様の効果が得られることになる． 図\ref{screenshot42}のように食料品店を選択した状態から 時間を置かずに▼を押すと， 下位層は展開されず，次の「衣料品店」が選択される．
ここで操作を止めて一定時間待つと 下位層が自動的に展開されて「靴屋」が選択される．
図\ref{screenshot8}の状態から▲を押すと， 下位層は自動的に閉じられて図\ref{screenshot7}の状態に戻る． さらに▲を押すと 「食料品店」の下位の層も閉じられ，図\ref{screenshot42}の状態に戻る． % また，図\ref{screenshot8}の状態から▼を2回押すと 「食料品店」の下位層は自動的に閉じられて図\ref{screenshot9}の状態になる． まとめると，
  • 選択した項目に下位層が存在するときキー入力を行なわずに待つと下位層が自動的に展開され，下位層の最初の項目が選択される
  • 項目リストの端を選択しているとき，さらに▲▼を押すと下位層は閉じられてひとつ上の層の項目が選択される
というふたつの工夫により， ▲と▼だけで 階層データを自由にナビゲーションすることが可能になっている．

Gearの利点

入力装置は単純であるにこしたことはありませんが、 2個のセンサであらゆる操作が可能なのであれば 左右の回転操作や 何かを押したり引いたりといった 単純な装置でも操作が可能だということになるので、 様々な環境で利用が可能だということになります。
{\GEAR}の操作は{\up}{\down}というふたつの入力しか必要としないため， 圧力センサや回転ダイヤルなどを利用した各種の実装が可能である． % 図\ref{disk}は，回転円板による{\GEAR}の実装である． 左に回転したとき{\up}が出力され， 右に回転したとき{\down}が出力されるようになっており， 左右に回転するだけでコンテンツを選択できるようになっている．
図\ref{paddle}は， パドルに貼った2個の圧力センサの値を{\up}{\down}に割り当てている．
このように，現状のGUIでは利用されていないようなデバイスでも {\GEAR}の入力装置として利用することができる．

Gearの利用環境

Gearの操作には2個の入力装置しか必要としないので、 例えば食卓の裏にさりげなくスイッチや圧力センサをふたつ並べておけば、 それらをさりげなく操作するだけでBGMを変えることができるでしょう。 テレビの前のソファの手すりにローラーをつけておけば、 それを回すだけであらゆるコンテンツを選択することができます。 私は自宅の机の前のモニタで常にGearブラウザを動かしており、 マウスホイールを使ってコンテンツを選択して視聴しています。 パソコンで音楽や動画を再生しながら仕事をしている人は多いと思いますが、 手近にGear端末があれば マウスホイールを回すだけであらゆるコンテンツを簡単に視聴できるのはとても快適です。 沢山のボタンを必要とする入力装置では どうしてもボタンにあわせた設計が必要になりますが、 Gearの場合には そもそもどこでコンテンツを視聴したいのかを最初に考え、 その場所に最も適した入力装置の形態を考え、 それらをGearのコントローラに利用するとく順番でデザインができることになります。 これまでの電子機器は 機械の都合にあわせて装置の形状がデザインされるのが普通でしたが、 それを根本的に改善できる可能性があるといえるでしょう。
